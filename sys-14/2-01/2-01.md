### Кейс 1
- Все зависит от выделенных средств на ПО для серверов)
- Если деньги на ПО есть, для сервера контроля доступа и управления учетными записями я бы выбран ОС Windows Server с ролью Active Directory Domain Services, так как она позволяет достаточно просто управлять учетными записями и их правами. Так же если на пользовательских ПК так же стоит ОС Windows, доменная структура позволяет гибко управлять инфраструктурой через групповые политики. Думаю что что-то похожее можно организовать и на Linux, но это более трудозатратно.
- Для терминального сервера я бы тоже выбрал ОС Windows Server с ролью Службы удаленных рабочих столов. Так же терминальный сервер можно организовать и на Linux, примеры таких решений видел в интернете).
- Для серверов веб-приложений лучше всего подойдет ОС на основе Linux, так как для она более оптимально подходит для этих целей
- Для прокси сервера так же выбрал бы ОС на основе Linux
